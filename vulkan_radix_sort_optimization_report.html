vulkan_radix_sort 优化算法图解报告

分析 jaesung-cs/vulkan_radix_sort 的 GPU radix sort：它如何把一轮 8-bit sort 拆成 upsweep → spine → downsweep，并通过局部 histogram、prefix scan、shared-memory 重排和 ping-pong buffer 提升吞吐。

RADIX = 256 WORKGROUP_SIZE = 512 PARTITION_SIZE = 4096 4 passes × 8 bits Vulkan compute

一句话总结

这个实现快的关键不是“直接 parallel scatter”，而是先把全局排序拆成 partition 局部问题，再用 prefix scan 合成全局偏移，最后在 downsweep 中先用 shared memory 把同 radix 元素聚合，让 global write 更接近连续。

pos ≈ globalBucketBase[d] + partitionPrefix[p][d] + localRankWithinPartition[d]

已测性能背景

keys-only, N = 2²⁰~0.715 ms
key-value, N = 2²⁰~0.799 ms
相对 VkRadixSort multi~2.4×

硬件记录：NVIDIA Tegra Thor；结果来自本地 benchmark 观察。

Upsweep：局部 histogram 降低 global atomic 压力

朴素做法

每个元素直接打 global atomic

atomic_add(globalHistogram[digit], 1)

大量线程竞争同一组全局 bucket，global atomic 压力高。

jaesung 做法

先打 shared local histogram

atomic_add(localHistogram[digit], 1)

竞争被限制在 workgroup 内，shared atomic 通常更便宜。

写出

每 partition 每 bucket 写一次

partitionHistogram[p][d] = localHistogram[d]

全局写入规模变成 partitionCount × 256。

Spine：扫描 partitionHistogram 得到全局 scatter 偏移

Spine 阶段固定 dispatch 256 个 workgroup：一个 radix bucket 一个 workgroup。每个 workgroup 扫描所有 partition 在该 bucket 上的计数。

真实代码中使用 wave-level prefix sum 来做这些 scan，减少显式循环和同步开销。

Downsweep：稳定 scatter 的关键公式

dst = globalHistogramPrefix[digit] + partitionHistogramPrefix[partition][digit] + localRankWithinPartition[digit]

优化核心是如何高效得到 localRankWithinPartition，以及如何让最终 global write 更连续。

1. Wave ballot

用 wave ballot 找出同一个 wave 中 digit 相同的 lane。

mask = WaveActiveBallot(sameDigit)

2. Wave 内 rank

计算当前 lane 前面有多少个相同 digit 的元素。

waveOffset = popcount(mask & laneMask)

3. 代表线程 atomic

同一个 wave 内同 digit 只选代表去更新 shared histogram。

if waveOffset == 0: atomic_add(...)

优化点卡片

01

大 partition

4096 elements/workgroup，减少 partition 数量和 spine scan 长度。

02

local histogram

把每元素 global atomic 替换成 shared memory 局部统计。

03

partition histogram

记录每个 partition 的 bucket 计数，让全局 scatter 可分解。

04

per-radix spine scan

固定 256 个 workgroup，每个 bucket 独立扫描所有 partition。

05

wave ballot rank

用 subgroup 操作计算 wave 内相同 digit 的 local rank。

06

representative atomic

同 wave 同 digit 只由代表线程 atomic，减少 shared atomic 次数。

07

shared rearrange

先在 shared memory 按 digit 重排，再连续写出，是 downsweep 的关键优化。

08

key-only / key-value 分离

key-only shader 不承担 value 搬运成本；key-value 才启用额外路径。

09

ping-pong buffers

每轮 input → output，然后交换 descriptor，避免 in-place scatter 冲突。

性能诊断摘要

瓶颈类型memory/scatter dominated
主要杠杆write coalescing
正确性门槛stable order

radix sort 的算术很少，吞吐通常受内存访问、atomic、barrier 和 scatter 地址形态影响。

Trade-off

优点：高吞吐、无 CPU readback、适合 GPU pipeline 串接、key-only 路径干净。

代价：shader 复杂、shared memory 用量高、依赖 wave/subgroup、每轮 3 个 dispatch + barrier，小 N 不一定划算。

  • src/shader/upsweep.slang：partition local histogram，写 partitionHistogram 和 globalHistogram。
  • src/shader/spine.slang：dispatch(RADIX,1,1)，每个 radix workgroup 扫描 partition histogram。
  • src/shader/downsweep.slang：wave ballot、local rank、多级 shared scan、shared memory rearrange、global scatter。
  • src/vk_radix_sort.h.in：host 端 4-pass loop、upsweep/spine/downsweep dispatch、barrier、descriptor ping-pong。
